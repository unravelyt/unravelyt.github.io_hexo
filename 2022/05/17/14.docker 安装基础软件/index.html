4 安装 Redis

  1. 拉取镜像

    docker pull redis:latest

  2. 配置文件

    *注释掉bind 127.0.0.1, 或者修改bind 0.0.0.0,表示允许所有ip地址访问

    *修改peotected-mode yes

    改为：protected-mode no.

    protected-mode参数是为了禁止外网访问redis,如果启用了，
    则只能够通过localhost ip (127.0.0.1)访问Redis

    daemonize no #用守护线程的方式启动

    requirepass root #给redis设置密码 密码要复杂，不然就会被黑

    appendonly yes #redis持久化　　默认是no

    tcp-keepalive 300 #防止出现远程主机强迫关闭了一个现有的连接的错误 默认是300

  3. 执行run命令


5 安装 wurstmeister/kafka （推荐）

  1. 拉取镜像

    docker pull zookeeper:3.8.0
    docker pull wurstmeister/kafka:2.13-2.8.1

  2. 启动 zookeeper

-p 2888:2888 -p 3888:3888

1、2181：对cline端提供服务

2、3888：选举leader使用

3、2888：集群内机器通讯使用（Leader监听此端口）

  1. 启动 kafka

参数说明：

-e KAFKA_BROKER_ID=0
在kafka集群中，每个kafka都有一个BROKER_ID来区分自己

-e KAFKA_CREATE_TOPICS=”topic-test:2:1”
topic-test1有2个分区1个副本，副本必须要有一个，1个表示没有备份的，
如果设置topic那么就不会自动生成了

-e KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181
配置zookeeper管理kafka的路径172.16.0.13:2181/kafka

-e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://172.16.0.13:9092
把kafka的地址端口注册给zookeeper，如果是远程访问要改成外网IP,类如Java程序访问出现无法连接。

-e KAFKA_LISTENERS=PLAINTEXT://0.0.0.0:9092 配置kafka的监听端口
